## 📁 Project Overview

This repository is organized into two distinct sections:

### 1. Exception Handling

These tasks were all about "defensive programming." Instead of letting the code break, I learned how to anticipate errors and handle them gracefully.

* **Key Skills:** `try-except-finally` blocks, custom error raising (`raise`), and managing specific errors like `ValueError`, `ZeroDivisionError`, `FileNotFoundError`, and `PermissionError`.

### 2. Object-Oriented Programming (OOP)

This section focused on building scalable, clean applications using core OOP principles.

* **Key Concepts:**
* **Classes & Objects:** Defining blueprints for real-world entities.
* **Encapsulation:** Protecting data using private attributes (`__price`) and setter methods to enforce rules.
* **Inheritance & Overriding:** Extending parent classes to create specialized objects while reusing logic.
* **Polymorphism:** Writing flexible code that can handle different object types through a common interface.
* **Abstraction:** Using Abstract Base Classes (`ABC`) to enforce a contract for subclasses.
* **Magic Methods:** Overloading operators (`__add__`) and string representations (`__str__`) to make objects behave more like native Python types.
